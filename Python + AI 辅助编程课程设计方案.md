# Python + AI 辅助编程课程设计方案

## 课程整体架构

本课程为期两周，旨在通过 Python 基础语法教学与 AI 辅助编程实践，培养学生的计算思维和问题解决能力。课程采用「理论学习 + 工具使用 + 实战项目」的递进式教学设计，让学生在两周内完成从 Python 入门到独立部署个人网站的全流程。

```
┌─────────────────────────────────────────────────────────────┐
│                    两周课程完整规划                           │
├─────────────────────────────────────────────────────────────┤
│  第一周                    │  第二周                        │
│  ┌─────────────────┐      │  ┌─────────────────┐          │
│  │ Python 语法入门   │      │  │ 数学部分        │          │
│  │ + 工具使用       │      │  │ + 网站完善      │          │
│  │ + 科研分享       │      │  │                 │          │
```

## 第一周课程详情

### 1.1 课程时间分配（120 分钟）

| 时间段 | 内容 | 执行者 | 准备材料 |
|--------|------|--------|----------|
| 0—60 分钟 | Python 核心语法（英文视频 + 中文注释讲解） | 中文讲师 | 已录制的 Python 语法视频、注释文档 |
| 60—90 分钟 | 工具演示 + Flask 网站创建 | 助教 | Tencent Cloud Studio 账号、演示代码、提示词文档 |
| 90—120 分钟 | Spotlight 科研分享 | 高校研究员 | 分享 PPT 或视频 |

### 1.2 课程内容详述

#### 1.2.1 Python 核心语法讲解（60 分钟）

本环节由中文讲师播放预先录制的 Python 核心语法课程（英文原版），边播放边进行实时注释讲解，确保学生理解每一个语法概念。讲解重点包括：变量与数据类型、条件判断与循环语句、函数定义与调用、列表与字典等常用数据结构。讲解过程中设置三个答疑时段，每 20 分钟一次，允许学生提问并由讲师当场解答。

#### 1.2.2 助教演示环节（30 分钟）

本环节由助教现场演示两个核心任务，帮助学生掌握后续作业所需的技术工具。

**第一部分：Tencent Cloud Studio 使用（10 分钟）**

| 演示项目 | 操作步骤 |
|----------|----------|
| 登录平台 | 演示如何访问 Tencent Cloud Studio 并使用教育邮箱登录 |
| 查看作业 | 演示在作业列表中找到本周任务，了解作业要求和截止日期 |
| 提交作业 | 演示上传代码文件、填写作业说明、提交作业的完整流程 |
| CodeBuddy AI | 演示如何打开 AI 助手、输入问题、查看 AI 回复 |

**第二部分：Flask 网站创建演示（20 分钟）**

| 演示阶段 | 内容 | 时长 |
|----------|------|------|
| 第一步演示 | 打开 CodeBuddy AI，粘贴「角色初始化」提示词，展示 AI 如何以问答形式收集需求 | 5 分钟 |
| 第二步演示 | 粘贴脚手架代码，要求 AI 逐行解释每个 Python 函数的作用 | 5 分钟 |
| 第三步演示 | 粘贴「模块适配」提示词，展示如何询问添加功能的方法 | 5 分钟 |
| 第四步演示 | 演示如何让 AI 写出代码，并要求 AI 在报错时解释原因而非直接给答案 | 5 分钟 |

#### 1.2.3 Spotlight 科研分享（30 分钟）

邀请高校研究员分享个人科研方向，帮助学生了解计算机科学在实际研究中的应用场景。分享内容可包括：人工智能在医学影像诊断中的应用、机器学习算法在金融风控中的实践、数据科学在社会科学研究中的案例等。分享结束后设置 10 分钟 Q&A 环节，鼓励学生提问。

## AI 辅助编程四步法

本方法引导学生将 AI 视为「对话式顾问」而非「答案生成器」，通过系统化的四步流程培养独立思考和代码理解能力。

### 3.1 第一步：角色初始化与「面试」启动

**目的：** 建立信心，让 AI 进入顾问模式而非写代码模式。每次只问一个问题能防止信息过载，让学生在回答中理清自己的逻辑。

**学生输入内容（直接复制使用）：**

```
你是一个资深 Web 全栈架构师，我是一名初中生。我想用 Flask 框架做一个个人网站，展示我的介绍、成就和爱好。请你以多轮讨论的形式，每次只问我一个问题，告诉我你需要什么信息来帮我设计这个网站？
```

**预期 AI 响应：** AI 会询问一个关于网站需求的问题，例如「你希望网站的整体风格是简约现代还是活泼有趣？」

**学生任务：** 认真思考并回答 AI 的问题，保持至少 3 轮问答交流。

---

### 3.2 第二步：脚手架代码注入与代码追踪

**目的：** 建立「代码所有权」意识。要求 AI 逐行解释代码，能培养学生的代码阅读和追踪能力，这是学习编程的基础。

**学生输入内容（先粘贴脚手架代码，再输入以下内容）：**

```
现在我已经准备好了一个基础的 Flask 网站脚手架代码（见下文）。请你逐行分析这段代码，并用我能听懂的语言解释每一个 Python 函数的作用。
```

**预设脚手架代码：**

```python
from flask import Flask, render_template

app = Flask(__name__)

# 个人信息数据
profile = {
    "name": "小明",
    "avatar": "/static/images/avatar.jpg",
    "bio": "热爱编程的初中生",
    "hobbies": ["编程", "钢琴", "篮球"],
    "achievements": ["校编程比赛一等奖", "钢琴八级"]
}

@app.route('/')
def index():
    """首页路由 —— 显示个人主页"""
    return render_template('index.html', profile=profile)

@app.route('/about')
def about():
    """关于页面路由"""
    return render_template('about.html', profile=profile)

@app.route('/contact')
def contact():
    """联系页面路由"""
    return render_template('contact.html')

if __name__ == '__main__':
    """启动 Flask 开发服务器"""
    app.run(debug=True, host='0.0.0.0', port=5000)
```

**学生任务：** 认真阅读 AI 的逐行解释，对于不理解的地方继续追问 AI。

---

### 3.3 第三步：需求比对与模块适配

**目的：** 引导通过「模式识别」来定位代码。研究发现，AI 在进行「模式转换」（将 A 模板改为 B 功能）时表现最稳健，且学生最易理解。

**学生输入内容：**

```
基于我们之前的讨论，以及这个脚手架代码。请你告诉我：

1. 为了实现「展示我的钢琴作品卡片」，我需要在这个脚手架中添加什么？

2. 为了让网站变成「星空风格」，我需要修改哪一部分 CSS？
```

**学生任务：** 记录 AI 的回答，理解需要修改哪些文件或添加哪些代码。

---

### 3.4 第四步：阶梯式迭代与日志记录

**目的：** 培养问题解决能力。要求 AI 在报错时先解释原因再教修复方法，而非直接给最终答案。

**学生输入内容：**

```
请帮我写出修改后的代码。如果代码运行报错，请解释报错信息的原因，并教我如何修复它，而不是直接给我最终答案。
```

**学生任务：**

- 按照 AI 的指导修改代码
- 如果遇到新错误，重新询问 AI
- 将每次交互记录到 AI 辅助日志中
